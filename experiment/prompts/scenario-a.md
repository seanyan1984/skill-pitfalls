# 场景 A：delegate_task 超时（webnovel-cross-validation）

## 错误描述

我在做人设维度的交叉验证分析。用 delegate_task batch 2 并发，每个子 agent 负责 2 本书。

batch2 的两个任务（AgentA_批3 + AgentA_批4）都超时了，600s 没写出任何文件。

batch2 的书目：
- 批3：反派(47k字) + 西游·虎力(47k字) = 94k
- 批4：两界(49k字) + 鹤形桩(52k字) = 101k

之前的 batch1 成功了（临圣+黑暗修仙=79k）。

请帮我诊断原因并给出修复方案。

## 期望的诊断

根因：人设维度分析密度高（逐章抠角色细节），2本/subagent 在高密度维度容易超时。之前金手指维度2本能跑是因为金手指场景稀疏。

期望修复：
1. 不要重试同配置
2. 拆成 1本/subagent 重跑失败批次
3. 已成功的 batch1 不需要重跑
