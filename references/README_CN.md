# Skill Pitfalls — 中文说明

## 这是什么

AI Agent 的 skill 文档里，PITFALL（踩坑记录）是最容易烂掉的部分。踩了坑就加一行，日积月累变成 30+ 条平铺列表，没分类没结构，agent 遇到报错时根本找不到对应的修复方案。

这个项目提出了一套结构化方案：分类 + 决策表 + 去重检查清单。

## 核心规则

1. **超过 5 条必须分类**（异常诊断 / DOM坑 / 反爬 / 数据 / 输出 / 项目约定）
2. **异常诊断类必须用决策表**（症状 → 判断 → 修复），不要写散文
3. **不跟正文重复**——正文已覆盖的坑，PITFALL 只给交叉引用
4. **追加前过检查清单**——分类 → 归属 → 决策表 → 去重 → 拆分

## 实验验证

我们做了 A/B 对比实验（3 个场景 × 2 个版本）：

| 指标 | 平铺列表（旧） | 分类+决策表（新） | 变化 |
|------|--------------|----------------|------|
| 综合分 | 2.67 | 3.67 | **+37%** |
| 修复准确度 | 1.67 | 2.0 | **+20%** |
| 推理步骤数 | 3.6 | 2.7 | **-25%** |

详见 `experiment/results/summary.md`。

核心发现：
- 信息散落在多处时差异最大（4 章节 24 条 → 1 个决策表，+2.0 分）
- 决策表消除歧义——旧版描述含糊导致 agent 理解偏差，新版修复列直接无歧义
- 决策表迫使诚实——匹配不完美时标注"部分匹配"而非强行套用

## 社区现状

详见 `references/community-landscape_cn.md`。

简单说：Hermes 官方、agentskills.io、Claude Code 社区都没有 PITFALL 内容级治理的规范。社区最关注的 skill 质量问题是文件系统层面的（路径断了/重复了），skill 内部内容组织是盲区。

## 怎么用

Hermes Agent 用户：
```bash
cp -r skill/ ~/.hermes/skills/software-development/pitfall-organization/
```

其他 Agent 框架：`skill/SKILL.md` 里的规则是框架无关的，直接参考即可。
