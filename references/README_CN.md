# Skill Pitfalls — 中文说明

AI Agent 的 skill 文档里，PITFALL（踩坑记录）是最容易烂掉的部分。踩了坑就加一行，日积月累变成 30+ 条平铺列表，agent 遇到报错根本找不到对应修复。

这个项目提出一套结构化方案：**分类 + 决策表 + 去重检查清单**。

## 实验结果

A/B 对比（3 场景 × 2 版本）：

| 指标 | 平铺列表 | 分类+决策表 | 变化 |
|------|---------|-----------|------|
| 综合分 | 2.67 | 3.67 | **+37%** |
| 修复准确度 | 1.67 | 2.0 | **+20%** |
| 推理步骤 | 3.6 | 2.7 | **-25%** |

核心发现：信息散落在多处时差异最大（4 章节 24 条 → 1 个决策表，+2.0 分）。

## 五条核心规则

1. **超过 5 条必须分类**（异常诊断 / DOM坑 / 反爬 / 数据 / 输出 / 项目约定）
2. **异常诊断必须用决策表**（症状 → 判断 → 修复），不写散文
3. **不跟正文重复**——正文已覆盖的，PITFALL 只给交叉引用
4. **追加前过检查清单**——分类 → 归属 → 决策表 → 去重 → 拆分
5. **重构后做质量审计**——7 维度打分

## 怎么用

Hermes Agent：
```bash
cp -r skill/ ~/.hermes/skills/software-development/pitfall-organization/
```

其他框架：`skill/SKILL.md` 里的规则是框架无关的，直接参考。
